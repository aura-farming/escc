# ESCC v1.7.1 — Full-run hardening patch

_Released 2026-07-06 · [CHANGELOG](../../CHANGELOG.md)_

A verification release: a **full run of the plugin** — a 66-check machine pass
(every `escc` CLI verb end-to-end in a hermetic home; all 28 hook matchers
fired through the real dispatcher with expected allow/block outcomes) plus
three parallel audits (cross-references, docs accuracy, adversarial bug hunt).
Everything found is fixed here; no new behavior surface.

## Fixed

- **SessionStart bootstrap version-skew** — the significant find. The
  bootstrap resolved the plugin root as env vars → `~/.claude` → installed
  marketplace/cache copies, **never its own tree**. In the exact case it
  exists for (SessionStart firing before `CLAUDE_PLUGIN_ROOT` is populated) it
  silently ran a **stale installed copy's** session hydration — reproduced
  live against a v1.5.0-era marketplace install. Self-resolution now beats any
  other install, with a regression test that proves the payload comes from the
  executing tree.
- **chaining-hints false positive**: a *contact* read at a company named
  "Dealify Inc" matched the substring filter and produced a deal-review hint.
  `input_match` is now a word-boundaried regex (objectType `deals`/`0-3` or a
  query containing the word *deal(s)*); Dealify-class inputs stay silent.
- **intent-router cache bleed** across plugin roots (cache now root-keyed).
- **enrichment-ops rule-path typo** (`rules/common/lawful-basis.md` →
  `rules/lawful-basis.md`) and a stale `commands-core` manifest count
  ("All 68" → 69).

## Docs & surface corrections

- **`.env.example` finally matches the code**: +22 undocumented user-facing
  vars (with real defaults), −4 vars that were documented but never read.
- **SKILL-DEVELOPMENT-GUIDE** documents the 220-char description cap + 14k
  total routing-surface pin (previously contributors hit the CI error blind).
- Getting-started guides list the **full persona modules** (SDR gains
  `enrichment-ops` + `outreach-analytics`; AE gains its six deal-maturity
  skills), and `contexts/watch-list.md` references now say it is
