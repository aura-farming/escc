# ESCC v1.5.0 — Per-account tone-match

_Released 2026-06-25 · [CHANGELOG](../../CHANGELOG.md) · [ADR-0015](../DECISIONS.md)_

Phase C of the Phase A→C roadmap set by ADR-0013. A draft no longer just sounds
like the **rep** — it can now sound like the **account** it is going to, while
every fact still comes only from approved product-knowledge.

## Highlights

- **Per-account voice overlay.** A deterministic, no-ML reader of how a specific
  account writes — formality, sentence length, question rate, greeting/sign-off,
  and their recurring vocabulary — layered on the rep's base `[VOICE PROFILE]`.
- **STYLE only, by construction.** The mirrored lexicon borrows the buyer's
  **words**, never their **claims or numbers**. A metric, percentage, or
  currency figure can *never* become a mirrored term, and no source sentence is
  echoed. Facts stay sourced only from approved product-knowledge.
- **No new dependency, no new trust boundary.** Pure libs + one operator CLI
  verb; buyer text reaches it only through the existing read-only
  quarantine/thread path.

## How to use

The buyer side is gathered by an orchestrator (a drafting skill, or
`transcript-analyzer` / `email-outbound-ops`) and passed as JSON — the rep's own
sent copy is never fed in:

```bash
escc voice account "domain:company.test" --input '{"texts":[
  "We need reporting and invoicing aligned before go-live.",
  "Invoicing accuracy is the priority for the rollout."
]}'
escc voice show "domain:company.test"
```

The overlay is then read by any drafting skill alongside the rep base profile.
A draft becomes: **rep base voice × buyer-role register × this-account register
× mirrored lexicon**. The base profile still wins on the rep's Banned/Preferred
Moves.

## The guarantee (why this is safe)

The style/content split (ADR-0013) is enforced **at write time, not by a
prompt**. A lexicon token survives only if it is pure-alphabetic
(`/^[a-z][a-z'-]*$/`) with stopwords removed, so a digit-bearing string can never
ride the mirroring channel back out as if it were our proof. The content guard
plants a buyer claim (`"cut invoicing costs by 47% and save $2.3M"`) and asserts
the rendered overlay carries neither the figure nor the sentence — only the
neutral vocabulary (`invoicing`, `reporting`, `rollout`).

## Upgrade notes

- **Nothing to migrate.** Overlays are per-account runtime data and live only in
  your gitignored workspace; existing installs gain the `escc voice` verb on
  update.
- **Version metadata reconciled to 1.5.0** across the package/plugin/marketplace
  manifests and the docs (`SOUL.md`, `AGENTS.md`, and `agent.yaml` were stale
  from earlier releases and are now current).
- **Catalog unchanged** (66 skills, 18 agents, 68 commands) — Phase C adds
  machinery and one CLI verb, no new skill / agent / command.

## Scope & deferred

- The register is computed from buyer text the orchestrator supplies; there is
  **no auto-mining of account-memory** (its events are mostly rep notes, which
  would mis-read the register).
- Greeting/sign-off detection is best-effort.
- Auto-inferred resonance and the outcome-fed ingestion loop remain **deferred**
  (ADR-0012).
