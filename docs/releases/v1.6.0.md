# ESCC v1.6.0 — The agentic routing core

_Released 2026-07-06 · [CHANGELOG](../../CHANGELOG.md) · [ADR-0016](../DECISIONS.md)_

ESCC now **auto-invokes reliably instead of waiting for slash commands**. The
release fixes the silent failure that made the plugin feel manual — the skill
catalog had outgrown the harness's routing budget, so most skills carried no
trigger text in context and could never auto-invoke — and adds the two hooks
that make ESCC propose the next move on its own.

## Highlights

- **Every skill is routable again.** All 66 descriptions compressed from
  39,193 to 12,645 chars (avg 594 → 192) into collision-aware trigger lines —
  and a CI pin guarantees the routing surface can never silently overflow
  again.
- **Prompt-time routing, budget-independent.** A deterministic
  **intent-router** hook matches what you typed ("they haven't replied…") and
  hints the right skill (`escc:follow-up-ops`) — even on small-context models
  where descriptions get truncated.
- **The next play, proposed.** A **chaining-hints** hook notices a high-signal
  tool result and suggests the chained skill: transcript → `discovery-notes`,
  Gmail thread → `reply-handling`, HubSpot deal read → `deal-review`. Once per
  family per session — never spam.
- **Nothing gained authority.** Both hooks are pure hints: they never block,
  never rewrite, fail open, and the fail-closed send-gate is untouched.

## What's new

| Surface | Change |
| --- | --- |
| `skills/*/SKILL.md` (all 66) | Descriptions compressed to ≤220-char trigger lines; 5 collision clusters (forecast / follow-up / call / pipeline / deal) resolved with unique phrase ownership. Detail unchanged in each body's "When to Activate". |
| `scripts/ci/validate-skills.js` | Routing-budget pin: description >220 chars or total >14,000 chars fails the build. |
| `prompt:intent-router` + `config/skill-keywords.json` | ~60 priority-ordered routes (compliance first, specific before general) → ONE `escc:<skill>` hint per prompt. Skips slash commands, explicit `escc:` mentions, short prompts. |
| `post:chaining-hints` + `config/tool-skill-chains.json` | 3 high-precision chains with per-session family dedupe and an `input_match` filter (only *deal* CRM reads hint `deal-review`). Errored calls skipped. |
| `scripts/hooks/session-start.js` | Startup-only `/daily` nudge (lowest-priority block — first dropped under budget). |
| `docs/HOOKS.md` · `hooks/hooks.json` · `run-with-flags.js` | Both hooks documented and wired; `prompt:` hookId prefix maps to UserPromptSubmit. Catalog: **28 hook matchers** (was 26). |

## How it feels

```text
you: they replied to my pricing email, how should I respond?
     🧭 escc intent-router: this looks like a job for the escc:reply-handling
        skill (/reply) — disposition the inbound reply … (ignore if it misreads)

you: pull the open deals for Acme
     [HubSpot read runs]
     🔗 escc next-step: you just pulled deal data — escc:deal-review scores
        MEDDPICC health with gaps as dated actions …
```

## The guarantee (why this is safe)

The hints are **data-driven and advisory**: routing tables are JSON config,
matching is deterministic regex (no model call), each hook injects at most one
line and says "ignore if this misreads the ask" — the model decides, the hook
never does. Every enforcement surface — the fail-closed send-gate, CRM write
guard, compliance protection, attachment quarantine — is byte-for-byte
untouched. Unit tests assert every route and chain points at a real skill, and
the collision clusters have regression tests.

## Fixed

- A **date-bomb** in the send-gate's clean-path test: it approved a draft at a
  pinned past date, so its 7-day approval token expired on 2026-06-30 and the
  (correctly behaving) gate started blocking it. The test now approves at the
  real current time — the machinery was never wrong.

## Upgrade notes

- **Nothing to migrate.** Existing installs gain the two hooks and the
  compressed descriptions on update; behavior of every command and skill is
  unchanged.
- The hooks run under the `standard` (default) and `strict` profiles; the
  `minimal` profile omits them. Disable individually with
  `ESCC_DISABLED_HOOKS=prompt:intent-router,post:chaining-hints`.
- Catalog: 66 skills, 18 agents, 68 commands (unchanged); 28 hook matchers.
