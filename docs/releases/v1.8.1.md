# v1.8.1 — public-source hygiene: nothing real can ship

**Type:** patch (security/hygiene) · **Behavior surfaces:** unchanged ·
**Catalog:** 68 skills, 18 agents, 70 commands, 30 hook matchers (unchanged)

ESCC is a public plugin: anyone can install it, and everything committed —
docs, examples, tests, even internal scaffolding — is published. v1.8.1 is
the result of a full-repo sensitivity audit (working tree, docs, machinery,
tests, GitHub release bodies, and git history) and closes every finding with
a guard that makes its class un-regressable.

## What was found and fixed

- **A personal filesystem path in a root doc.** An internal research document
  at the repo root carried the maintainer's dev-checkout path and local
  environment notes. The personal-path scanner never saw it: it scanned from
  a directory allowlist that missed root-level files, and two scaffolding
  directories were prefix-exempt from every scanner. The doc is removed; the
  scanners now cover every git-tracked file with no directory exemptions.
- **Real-registered domains posing as fixtures.** Example identities used
  domains that actually exist and are owned by someone (`company.com`, `x.com`,
  `startup.io`, …) — indistinguishable at a glance from leaked customer
  data, and a copy-paste hazard. Every fixture domain and email in the repo
  now uses an IANA-reserved TLD (`.example` / `.test`) that cannot resolve or
  receive mail; example CRM ids are abstract (`company:<hubspot-id>`); the
  canonical fictional company is **Example Co**.
- **Origin-identifying residue.** The v1.3.0 de-companyfication renamed the
  maintainer's employer but left its product-domain vocabulary readable in
  generic machinery and fixtures (benefit-signal keywords, outbound/voice
  test copy, example knowledge entries, one release doc), one live-workspace
  incident count in historical notes and a config comment, a real-session
  provenance note in a test header, and the employer name verbatim in a
  validator comment and the banned-token config. All neutralized.
- **Public release bodies.** The v1.8.0 and v1.3.0 GitHub release notes were
  re-edited to the same standard (release bodies live outside the repo, so no
  committed validator ever scanned them).

## The guards (what prevents recurrence)

1. **Committed = scanned.** `validate-no-personal-paths.js`,
   `validate-no-secrets.js`, `validate-no-company-tokens.js`, and the new
   email guard all enumerate `git ls-files` — root files, examples,
   `.claude/`, `.github/`, and docs scaffolding included. A scope-pin test
   fails the build if any scanner ever grows a directory allowlist or a
   prefix exemption again.
2. **`validate-committed-emails.js` (new).** Every email address in a
   committed file must use a reserved-TLD domain (or a subdomain of one) or a
   vendor notification address explicitly listed in
   `config/committed-email-domains.json` (currently: `hubspot.com`,
   `linkedin.com`). A real address fails CI; admitting a new fixture domain
   is a deliberate, reviewable config change.
3. **Hashed banned-token list.** `config/banned-company-tokens.json` now
   ships `sha256` hashes, so the public source no longer discloses the very
   name the guard exists to keep out. The scanner hashes every word-shaped
   and host-shaped candidate in every committed file (boundary-safe: a short
   brand never trips on a longer word; a banned host also covers emails at
   that host). Forks may still use the plaintext `tokens` array.
4. **Fixture conventions are now explicit.** Fixture personas (Jane, Sam,
   Jordan, Morgan), fictional companies on reserved domains
   (`company.example`, `sample.test`, Example Co), abstract ids — so an example
   can never be mistaken for, or collide with, anything real.

## Known-and-accepted (not changed here)

- The MIT license author line (`Copyright (c) 2026 Lucas`) and the ECC
  attribution (Affaan Mustafa) are deliberate, license-required authorship.
- `rules/jurisdictions/au.md` (AU Spam Act first-class) is product design,
  not a leak.
- **Git history before v1.3.0 still contains pre-scrub content, and commit
  author metadata carries the maintainer's work email** (ADR-0013 chose not
  to rewrite history). Cleaning that requires a history rewrite + force-push
  + re-tagging — a separate, deliberate operation tracked outside this
  release.
