# ESCC v1.3.0 — Open-source readiness (company-neutral by construction)

_Released 2026-06-25 · [CHANGELOG](../../CHANGELOG.md) · [ADR-0013](../DECISIONS.md)_

Phase A+E of the Phase A→C roadmap. ESCC is prepared for public release for any
sales team worldwide: it is **company-neutral by construction, enforced by CI** —
not by reviewer diligence. Any team can install it and keep their own data in
their gitignored workspace.

## Highlights

- **Generic by default.** The shipped controlled vocabulary is a cross-industry
  template — no competitors, no company-specific segments, a generic title map.
- **Your data lives in a per-workspace override**, seeded and grown with new CLI
  verbs — and it survives a plugin update.
- **Two CI guards make a leak un-regressable**: a brand name or a credential can
  no longer be committed without failing `npm test`.

## What's new

| Surface | Change |
| --- | --- |
| Per-workspace vocab override | `loadVocab` gains a workspace tier (`<data-home>/escc/product/knowledge-vocab.json`) between an explicit path and the shipped template, plus `escc product vocab show \| init \| suggest`. New `product-knowledge` helpers: `vocabSource`, `workspaceVocabPath`, `readVocabFile`, `slugifySegment`, `suggestSegments`, `initWorkspaceVocab`, `addSegmentsToWorkspace`; optional `note` field on the vocab schema. |
| CI guards | `validate-no-company-tokens.js` (banned-brand list, word-boundary matched, git-tracked-only) + `validate-no-secrets.js` (high-confidence credential signatures) join `npm test`. |
| Company-neutral defaults | `config/knowledge-vocab.json` ships generic (`competitors: []`, `segments: ["general"]`, cross-industry roles + title map); all committed example/seed/test data genericized (`Acme` / `competitor-x` / `Example Operator`). |
| `.gitignore` | Now also excludes runtime `voice/`, `patterns/`, and learned/pending instinct stores. |

## How to use

```bash
escc product vocab show           # inspect the active vocab + its source
escc product vocab init           # copy the generic template into your gitignored workspace override
# edit it to add YOUR competitors / segments / titles
escc product vocab suggest --input '{"industries":[...]}'   # grow segments from CRM industry values
```

## The guarantee

A brand name or a credential can no longer be committed without failing
`npm test`. The validators scan **only git-tracked files**, so they never
false-fail on gitignored runtime data, and the company-token match is
**word-boundary** anchored so it never trips on a substring (e.g. a brand inside
"standard"). Legitimate authorship — the MIT `LICENSE` / `plugin.json` author —
is **kept**; it is not a company token.

## Upgrade notes

- **Re-feed your own vocabulary.** The shipped vocab is intentionally empty of
  competitors/segments — no company data is migrated into the repo. Seed your
  own with `escc product vocab init` (then edit), or via `/ingest` (v1.4.0). Your
  override is gitignored and survives `plugin update`.

## Scope & deferred

- **Git history is left intact and scrubbed going forward.** No credential was
  ever committed (working tree or history), so a destructive `git filter-repo`
  rewrite — which would break every commit SHA, all merged PRs, and all clones —
  buys no security and is rejected.
