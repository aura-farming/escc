# v1.10.0 — the A-Z account attack-plan, plus review-path hardening

**Type:** minor (new capability + fixes) · **Governing decision:** ADR-0021 ·
**Catalog:** 69 skills, 18 agents, 71 commands, 30 hook matchers (this release
adds one skill + one command; agents and hooks unchanged)

## Why

Two threads land here. First, a rep asking *"research this business and give me a
plan of attack"* had no single surface — `account-research` produced a brief and
stopped, `prospecting-pipeline` worked a list not one deep named target, nothing
emitted a sequenced plan, and the phrasing routed nowhere. Second, a full-repo
review (15 lenses, adversarially verified) surfaced four small but real hardening
items in the outbound review path and the CLI.

## What shipped

### The A-Z on-ramp: `account-attack-plan` (`/attack`) — ADR-0021

Name one target business and get **both** a deep, sourced research brief **and a
sequenced, multi-channel plan of attack** in one pass:

- **Pre-flight first.** Resolves the canonical account (ADR-0018) and screens
  do-not-contact / contactability **before any research spend** — you never plan
  an attack on a blocked account, an existing customer, or another rep's deal.
- **Parallel research fan-out** over the existing read-only agents —
  `account-researcher` (identity, structure, footprint, size, money, stack,
  triggers), `prospect-researcher` (the buying committee), `competitor-analyst`
  (incumbent + displacement), `warm-path-mapper` (warm paths), `signal-scorer`
  (ICP fit + tier) — graded against an explicit **A-P rubric**; every uncovered
  dimension is stated, never invented; all fetched content is untrusted.
- **A real plan of attack**, synthesized by `sales-planner`: the entry point (who
  first, which channel, why now), the sequenced touches with the **approved
  proof** (`product-knowledge` only) and one CTA each, dates, objection pre-empts,
  the multi-thread widen, and a fallback if the entry point goes silent.
- **Draft-only, gated hand-off.** First touches route through
  `cold-outreach`/`outreach-drafter` → `outbound-reviewer` → `escc outbound
  approve` → the fail-closed send-gate, or stage the whole sequence via
  `/escc-worklist`. Open loops persist to `account-memory`; `crm-operator` stays
  the sole writer. **Nothing sends.**

It **orchestrates existing agents rather than adding one** — the strongest
research is a parallel fan-out of specialists (ADR-0021).

### Review-path hardening (four fixes)

- **Account-scope do-not-contact is enforced at the send-gate.** A per-recipient
  approval token minted *before* an account was blocked no longer outlives the
  block — the gate now re-derives the recipient's canonical account key (ADR-0018)
  and blocks on an active account-scope suppression, not just a contact-scope one.
  Tightening-only, best-effort (a resolution error can never break the gate).
- **`findValidApproval` fails closed on a non-numeric confidence** — a NaN/missing
  confidence is now rejected, matching the review-attestation path (was fail-open).
- **`escc doctor` tells the truth on a plugin install.** With no `escc install`
  file-copy targets it no longer prints "0 ok, 0 error(s)" as if healthy — it says
  it checked nothing and points to `/plugin`; under `--exit-code` it returns
  non-zero. Relevant to marketplace-install smoke tests.
- **Intent-router precision.** The worklist `\bbatch\b` route no longer hijacks a
  bare non-outbound "batch" (e.g. "batch of transcripts"); an outbound "batch
  prospect/draft/…" still routes to the worklist on-ramp.

## Review hardening, wave 2 (the resumed review + the loose-ends pass)

The interrupted review was resumed to completion (live-execution batteries,
docs-vs-code, all four persona use-case lenses — AE/Manager/RevOps came back
clean), then a final connectivity/loose-ends pass closed everything actionable.
Highlights (full detail in the CHANGELOG `[1.10.0]` entry):

- **Inbound opt-outs are now hook-enforced**: the new `escc dnc` verb writes the
  local blocklist the fail-closed send-gate reads, `opt-out-handling` runs it
  FIRST (before CRM suppression), and `dnc clear` refuses without documented
  re-consent evidence.
- **SECURITY**: `compliance-protection` now matches protected compliance files
  case-insensitively (macOS/Windows bypass closed).
- **`mcp-health-check` probe fixed**: spec-compliant stdio servers (exit-0 on
  stdin EOF) were misread as failed probes and blocked; the probe now speaks
  JSON-RPC `initialize` and treats an answer as healthy.
- **Installability**: `skills/worklist` (missing since v1.9.0) and
  `skills/account-attack-plan` were claimed by no install module — profile
  installs never shipped them. Both are placed, and a new manifest completeness
  guard makes the gap class impossible to reintroduce.
- **Four new/extended CI guards**: version-consistency across the 7 release
  surfaces, the 800-line machinery cap (dispatcher + query layer split back
  under it), manifest disk→module completeness, and an emoji drift ratchet.
- **Routing + CLI correctness**: the `/ingest`-vs-`quote-desk` shadow, four
  disambiguated routing pairs, inline-JSON `--input`, `--json` output modes,
  outcome `--type` guarding, `product add` shape validation.
