# ESCC v1.7.0 — Setup doctor + enrichment + funnel gaps closed

_Released 2026-07-06 · [CHANGELOG](../../CHANGELOG.md) · [ADR-0017](../DECISIONS.md)_

The follow-through to v1.6.0's routing core: **first-run can no longer fail
silently**, and the capability gaps the audit named are closed — inside the
existing safety invariants, not around them.

## Highlights

- **Setup is a doctor now.** `configure-escc` checks the MCP stack against
  reality (HubSpot/Gmail required — a missing one is called out with the fix,
  never silently passed), verifies the install with `escc doctor`, and offers
  the first-run seedings that make ESCC yours: vocabulary, voice via
  `/ingest`, and an optional persona routing focus.
- **Enrichment finally works.** `/enrich` (the new `enrichment-ops` skill)
  orchestrates whichever enrichment MCP is wired (Apollo/Clay, detected at
  runtime) with a research-agent web fallback — per-field provenance,
  `verified`/`reported`/`inferred` confidence labels, and review-pack-only
  writes via `crm-operator`. An inferred email is never a send target;
  unfilled beats invented.
- **Three funnel gaps closed in place:** renewal-window triggers
  (`trigger-detection` → `renewal-playbook`), a pending-approvals board
  (`deal-desk`), and a proactive referral ask (`follow-up-ops`).

## The guarantee (why this is safe)

`enrichment-ops` adds **zero new trust surface**: it is read-only +
propose-only (`crm-operator` remains the sole writer, ADR-0004), provider and
web output is untrusted data, PII collection is minimum-necessary
(`data-handling.md`), and no contact data is ever fabricated or
pattern-guessed. The setup doctor only *reads* tool availability; its one
optional write (`skillOverrides`) is shown as exact JSON first, is reversible,
and defaults to Skip. The renewal/approvals/referral additions ride existing
review and write paths.

## Upgrade notes

- **Nothing to migrate.** Existing installs gain `/enrich` and the wizard
  steps on update. Re-run "configure escc" any time to get the stack-health
  check and seeding pass on an existing install.
- Catalog: **67 skills, 18 agents, 69 commands** (CI-pinned); `skills-sdr`
  now carries 12 skills.
- Still out of scope (unchanged non-goals / no integration path): LinkedIn
  send-automation (no official API), eSignature (no MCP), comp modeling,
  win-loss interview automation.
